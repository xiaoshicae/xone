# 模块代码审查

对指定模块进行代码审查，分析潜在的优化点。

使用方法: /project:review <模块名>

示例:
- /project:review xhttp
- /project:review xgorm

$ARGUMENTS

---

请对 `$ARGUMENTS` 模块进行代码审查，分析以下方面：

1. **性能优化**
   - 是否有不必要的内存分配
   - 是否有可以预编译的正则表达式
   - 是否有可以缓存的计算结果

2. **并发安全**
   - 全局变量是否有适当的锁保护
   - 是否存在竞态条件

3. **代码质量**
   - 是否有重复代码可以提取
   - 是否有 magic string/number 应该提取为常量
   - 错误处理是否一致

4. **API 设计**
   - 是否缺少必要的公开 API
   - 是否与其他模块保持一致（如使用 xutil.ToDuration）

请以表格形式输出发现的问题和建议。
